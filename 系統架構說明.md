# 逍遙投資系統 — 架構與流程說明

## 一、系統概覽

- 本機：http://localhost:5000/
- 前台：https://tock-system.onrender.com/
- GitHub：https://github.com/ccyu1007-web/roger.git
- 本機 DB：SQLite（stocks.db）
- 雲端 DB：PostgreSQL（Render 提供）
- db.py：抽象層，自動切換 SQLite/PostgreSQL

## 二、資料來源優先級（重要！）

### 損益表（revenue/cost/gross_profit/operating_income 等）
**群益 > 政府API > Yahoo > FinMind**
- 群益直接覆蓋（最高優先級）
- Yahoo/FinMind 用 COALESCE 不覆蓋已有值

### 資產負債表（total_assets/total_equity/common_stock）
**群益 > Yahoo > FinMind**

### 現金流量表（operating_cf/capex）
**群益 > Yahoo > FinMind**

### 股利（cash_dividend/stock_dividend）
**群益 > FinMind**

### 月營收
**政府API（批次）> 群益（個股）> FinMind**

### 股價
**政府API（批次TWSE/TPEX）+ TWSE mis（即時API）**
- 批次API有日期驗證，不是今天就改用即時API

### 三大法人買賣超
**群益 zcl 頁面（每天17:10更新）**

### EPS/營收增率
**政府API（批次）> FinMind**

## 六、股價更新流程（關鍵！）

### refresh_prices() — 三段時間邏輯 + 日期驗證
1. 盤中（09:00~13:35）→ TWSE mis 即時API
2. 盤後同日（13:36~16:00）→ 仍用即時API（z值=收盤價）
3. 其餘時段 → 批次收盤API，但驗證資料日期
   - _twse_batch_date ≠ 今天且為平日 → 改用即時API

### run() 步驟 6b
- 寫入DB後檢查批次日期，不符就用即時API覆蓋

### 為什麼？
- TWSE批次API通常16:00後才更新，有時甚至18:00+仍未更新
- 14:30排程用批次API會抓到昨天的收盤價

## 七、年報資料來源寫入規則（關鍵！）

### 群益（capital_fetcher.py）
- 損益表欄位：**直接覆蓋**（excluded.field）
- 反算 operating_expense = gross_profit - operating_income
- 反算 non_operating 從季度累計

### FinMind（scraper.py）
- 損益表共用欄位：**COALESCE(existing, new)** 不覆蓋已有值
- FinMind獨有欄位：**COALESCE(new, existing)** 優先用新值

### Yahoo（yahoo_fetcher.py）
- 損益表已有就跳過
- 只補寫 total_assets/total_equity/operating_cf/capex

### 為什麼？
- FinMind 的 CostOfGoodsSold 分類與台灣財報不同（如台泥111年差552百萬）
- 群益的數據符合台灣財報標準

## 八、個股頁面載入流程

### 效能優化：先回舊資料，背景更新
1. DB 有資料 + 快取未過期 → 直接回傳（毫秒級）
2. DB 有資料 + 快取過期 → 先回傳舊資料，背景 thread 更新
3. DB 完全沒資料 → 同步抓（第一次才需等3-5秒）

### 快取時間
- 年報：24小時
- 季報：12小時
- 月營收：24小時
- PE歷史：7天

### 個股抓取流程（fetch_company_financials）
1. 群益 fetch_all_three()（損益表+資產負債表+現金流量表+股利+月營收+合約負債）
2. 檢查關鍵欄位是否齊全
3. 不足時才用 Yahoo 補充
4. 不用 FinMind（節省額度）

## 九、季度估計表衍生計算

### DB 沒有但 API 會反算的欄位
- 稅（所得稅費用）= 稅前淨利 - 稅後淨利
- 繼續營業單位損益 ≈ 稅後淨利
- 稅率 = 稅 / 稅前淨利
- 歸屬母公司權重 = 稅後淨利 / 繼續營業單位損益
- 每股盈餘-本業 = 營業利益 × (1-稅率) / 股數
- 每股盈餘-業外 = EPS - 本業EPS

## 十、總表動態欄位邏輯

### 季度EPS欄位
- 法定期限為基準（Q1→5/15, Q2→8/14, Q3→11/14, Q4→3/31）
- 如果實際資料有更新的季度（提前公告），以資料為準

### 年度EPS欄位
- 同上，資料驅動

### 股利欄位
- 有新股利資料就加欄，12月才剔除最舊一欄

## 十一、總表計算欄位

### 沈董EPS
- 用當年度已公佈季度推算全年（年化）

### 加權EPS（原名「權重EPS」）
- 五年加權平均（預設權重 30/25/20/15/10）

### 權重EPS（新增）
- 沈董EPS × 沈董權重% + 加權EPS × 加權權重%（預設各50%）

### 財務等級
- ROE + FCF比率 + 營業利益率 → 矩陣計算
- A級：紅色，B級：橘色，C以下：灰色

### 配息率
- EPS > 0：配息率 = 股利 / EPS × 100
- EPS ≤ 0 但有配息：直接顯示 100%

## 十四、換電腦還原

說「打開逍遙投資系統」，Claude Code 會自動：
1. 檢查 stock_system/ 是否存在 → 不存在就 git clone
2. 檢查 stocks.db 是否存在 → 不存在就從 iCloud 還原
3. 檢查套件 → 未安裝就 pip install
4. 檢查 Flask → 沒在跑就啟動
5. 開瀏覽器

## 十五、系統監控

### 熔斷器
- 異常率 > 10% → 熔斷，冷卻 10 分鐘
- 樣本 < 50 → 不熔斷

### 跳變校驗
- 股價單日跳變 > 2倍 → 攔截
- EPS跳變 > 10倍 → 攔截

### 交叉校驗
- 抽樣 20-30 支比對 DB vs 政府API
- 排程自動 + 手動觸發

### 每日報告
- 價位變動（便宜/昂貴區間移動）
- 財務等級升降
- ETF成分股異動
- 新聞分類（Tier 0/1/2）
